Firstly, Which one do I choose?

  • SimpleDS: use for maximum speed but long routes.
  • DistanceDS: minuscule decrease in speed compared to simpleDS but much shorter routes.
  • A-Star: use to find the shortest possible route. Slowest out of the three but still very fast. (RECOMMENDED!)

Modes? What are those?!

To simplify user interaction, I've synthesized two seperate systems of communicating commands to the computing architecture. In node mode, you interact with nodes while in path mode you interact with paths.

Instructions

The start of the journey is marked in yellow while the end is marked in green. If both are the same node, you will see an odd semicircle connection.

In Node Mode:
  • Drag and Drop a node to move it.
  • Press Left Mouse Button to create a new node.
  • Press DELETE or BACKSPACE over a node to delete it.

In Path Mode:
  • Drag and Drop from a node to another to create a path between each other.
  • Press Left Mouse Button to select a new node as the starting point or to mark it as the ending of a path.
  • Press DELETE or BACKSPACE to select a new node as the starting of a deletion or to mark it as the ending of a path deletion.

 In Both:
  • Press ESCAPE to toggle modes.
  • Press g to mark a node as the goal of the journey.
  • Press s to mark a node as the start of the journey.
  • Press r to toggle whether the route is shown or not.
  • Press n to toggle whether numbers on nodes.
  • Press c to clear screen and delete all nodes and paths.
